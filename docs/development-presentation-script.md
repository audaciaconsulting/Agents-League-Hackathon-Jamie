# Project Development Presentation Script

## Opening

This project had two goals: use Copilot to shape the work and use an Azure cloud AI service in the build.

The AI goal was to use Foundry to turn profile signals into game recommendations.

## Slide 0: Setting Up The Project

I started with the Awesome Copilot instruction files in this repo.

I also added a custom instruction so every prompt and response was logged in the prompt log, which kept the development history traceable.

## Slide 1: AI at the Core

The key design choice was to keep the AI honest.

The app only uses public data, then sends it to Foundry so the model can explain patterns, summarize the profile, and suggest games.

## Slide 2: Feeding Better Signals to AI

I focused on better inputs for the model.

The server started with Steam public profile data, then added recent play history when available for stronger recommendations.

## Slide 3: Making the AI Output Usable

The Angular client was built to make the response easy to read.

It shows recommendation cards, loading states, and the Foundry output once the analysis returns.

## Slide 4: Making the AI Setup Reproducible

The project also added Terraform-backed Azure provisioning so the Foundry environment can be recreated consistently.

The provisioning script reads `.env.local`, creates the Azure AI Services resources, and writes the connection details back for the app.

## Closing

In the end, this project built a reliable AI workflow from public signals to Foundry, and yes, the whole project, including this presentation script, was generated by AI.

You can find the prompt history for this presentation in the repository.
